Använd inte Gunicorn för Django och Flask på Heroku →

2014-04-24 av Anders Ytterström

Dave Hall skriver litet om varför man inte ska använda Gunicorn om ens Django-projekt ligger på Heroku.

Främsta orsaken är att Gunicorn är designat för att ligga bakom nginx, vilket inte är fallet på Heroku.

Jag kan också tipsa om django-herokuapp som innehåller en bootstrap i best practices. Jag kopierade mycket guldklimpar därifrån i mina aktiva Django-projekt.

Flask har samma bekymmer, dessvärre.
