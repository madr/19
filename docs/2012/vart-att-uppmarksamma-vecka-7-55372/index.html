Värt att uppmärksamma, vecka 7

  • A List Apart: Articles: Building Twitter Bootstrap
    Det här projektet är verkligen något att drömma om. Bootstrap är det förata CSS-biblioteket som fått godkänt under min lupp. Det är några riktigt skillade personer som gjort sitt yttersta i detta. Även en inspirerande artikel som är värd att läsa för att se hur hållbar och bulletproof CSS skrivs i verkligheten.
  • Coding Horror: Meetings: Where Work Goes to Die
    Ännu en artikel på ämnet "möten dödar produktivitet".
  • HTML5 Rocks - Getting Started With the Track Element
    Visa subtitles för videor enligt HTML5-standarden. Jag ser framför mig ljusa år för tillgänglighet och användbarhet i detta.
  • twolfson/File-Watcher - GitHub
    Verktyg som "övervakar" filer i ett webbprojekt och laddar om webbläsarfliken när någon av de övervakade filerna har ändrats. Kan vara anvndbart för t ex författande av enhetstester för JavaScript eller semi-live preview vid intensivt CSS-kodande.
  • Bricss - Forget loading spinners, use a wait cursor
    Inte så dumt. Det är mer native och mer universiellt. Önskar att det fanns något riktigt "loading"-state också, så att webbläsarnas egna spinners och loaders gick att tala med.
  • Building a High Performance Website » SitePoint¬es=
    Checklista på hur webbprestanda går till i praktiken.
  • Best Practice for Programming with Vendor Prefixes » SitePoint¬es=
    Mitt tips är att helt överge tanken på att hantera detta med JavaScript. Hantera CSS med klasser och applicera genom átt lägga till och ta bort klasserna med JavaScript.
  • Things It Might Be Fun/Useful to Try the Universal (*) Selector On | CSS-Tricks¬es=
    Sedan jag började med OOCSS har jag några utav dessa exempel i egna klasser, t ex .smooth for att enkelt kunna applicera transitions och spara några viktiga bytes i CSS-filen. Universal selector känns litet riskabelt och jag föredrar átt hålla dessa CSS-deklarationer sparsamma.
  • Every Time You Make a Good Business Decision, a Puppy Gets Cloned | Impressive Webs¬es=
    Rakt på sak. Det är faktiskt biz för de flesta av oss. Jag är själv tämligen religiös om användbarhet och att följa standarder, men jag tvingades kompromissa med min övertygelse redan första dagen jag började försörja mig som webbutvecklare.
  • html5sql.js Home Page
    Intressant idé som faktiskt kan bli nog så användbar vid höga krav på offline cache och enhetsspecifika parametrar.
  • A List Apart: Articles: Every Time You Call a Proprietary Feature "CSS3," a Kitten Dies
    Sant, så sant. Jag tänker dra mitt strå till stacken genom att uppmärksamma saker jag är medveten om är proprietärt i min kod, då jag inte är säker på att verktyg som CSS validator eller CSS lint hittar sådant.
